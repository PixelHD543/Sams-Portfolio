Autonomous Farming Equipment – Unreal Engine + ESP32 + LiDAR

Simulation: Unreal Engine 5 (self-pathing plugin)
Controller: ESP32 S3 (240MHz, WiFi/BT)
Sensor: RPLidar 2D (16K samples/sec)
Platform: 1/8th Nitro RC Car + Activity Bot

Project Overview – Modular Autonomous Farming

This BEng individual project developed a modular autonomous system that can be applied to any agricultural machinery. The system enables remote control and self-driving capability while leveraging British agricultural standards (all trailers share identical hookup ports).

Unreal Engine Operator Interface

Figure 1: Main Menu

The starting user interface with three main options: Settings, Running Operations, and Vehicles. From here, the user configures the system, loads operations, or adds new vehicles.

Figure 2: UI Hierarchy Diagram

When a button is clicked, all previous UI pages are removed and a new UI is created from the specific entry, then added to the user's view. This modular UI design keeps the interface clean and intuitive.

Figure 3: Operations Menu

All available operations are stored here, including the main calibration routine. The user selects which operation to run on which vehicle.

Figure 4: Vehicle Selection

After choosing an operation, the user selects the specific vehicle. A back button is available to return to the previous UI in case of user error.

Figure 5: Live Vehicle Telemetry

Once connected, Unreal Engine displays real-time vehicle information: distance to obstacle in front, steering angle, and throttle/brake state. Updates every 50ms.

Figure 6: Start Confirmation

After pressing "Start", the operation is loaded onto the microcontroller and a confirmation UI appears. The operation can be terminated at any time in an emergency.

Figure 7: Active Operation Screen

The vehicle in operation with system status set to "Active". The user can monitor all parameters and terminate the operation if needed.

Figure 8: Web Plugin Configuration

Unreal Engine's web browser plugin connects to the ESP32's HTTP server. The vehicle's IP address is set in the URL variable, enabling real-time data exchange.

Figure 9: Settings Menu

From the settings menu, the user can adjust vehicle parameters, add new operations, draw custom routes, or enable manual override for unexpected diversions.

Figure 10: Manual Override

Manual override mode allows the user to take control of any registered vehicle via Bluetooth controller – essential for navigating unmapped areas or handling emergencies.

Technical Deep Dive – LiDAR Path Planning

// LiDAR data processing in Unreal Engine
// Spawn oblong actors equal to LiDAR rotation + distance measurement
// 16,000 samples/sec → 16,000 boundary actors per second

function processLiDARScan(rotation, distance) {
  spawnBoundaryActor(rotation, distance);
  accumulatedScans++;
  if (accumulatedScans >= 16000) {
    updateNavigationMesh();  // Recalculate drivable area
    accumulatedScans = 0;
  }
}

// ESP32 Web Server endpoints

POST /drive       → { throttle: 75, steering: -30 }
POST /emergency   → { brake: 1 }
