Intel Quartus & DE10-Nano FPGA Development

FPGA: Intel Cyclone V (Terasic DE10-Nano)
HDL: Verilog / VHDL
Tools: Quartus Prime, ModelSim
Max Clock: 62 MHz (pipelined)

Project Overview

Developed multiple FPGA cores on the Terasic DE10-Nano board, focusing on real-time signal processing and embedded processor design. All projects version-controlled with Git and documented on GitHub.

  • ✅ 12-tap FIR filter with pipelined MAC – 62 MHz max frequency
  • ✅ Custom RISC-V RV32I soft-core processor (3-stage pipeline)
  • ✅ UART and SPI controllers for external sensor interfacing
  • ✅ 256-point FFT hardware accelerator – 8× faster than pure software

Technical Challenges & Solutions

Challenge 1: Timing closure – The FIR filter initially failed timing at 50 MHz due to long combinatorial paths in the multiplier-accumulator. Solution: Pipelined the MAC unit into 3 stages and added register banking, achieving 62 MHz with 12% logic utilisation.

Challenge 2: Memory bandwidth bottleneck – The RISC-V core stalled frequently on block RAM access. Solution: Implemented a wishbone bus with burst transfers and a 2KB direct-mapped instruction cache, reducing average memory access time by 68%.

Challenge 3: Debugging without oscilloscope – Signal probing inside Quartus Signal Tap logic analyzer. Created custom trigger conditions for UART receive errors.

Version Control & Hardware CI

All Quartus projects are managed in Git with this structure:

fpga-de10/
├── rtl/           # Verilog/VHDL source files
├── constraints/   # .sdc timing constraints
├── sim/           # ModelSim testbenches
├── quartus/       # .qpf project files
└── docs/          # Pin mappings, block diagrams

Git LFS tracks large binary files (.sof, .rbf). GitHub runs syntax checks on each push.
