CoppeliaSim Robotics Simulation + MATLAB Integration

Simulator: CoppeliaSim (formerly V-REP)
Control: MATLAB API / PID
Kinematics: Inverse & Forward
Target: 6-DOF robotic arm

Project Overview

Developed a complete robotics simulation pipeline connecting CoppeliaSim (physics engine) with MATLAB for control algorithm development. This project bridges the gap between simulation and real hardware.

  • ✅ Real-time MATLAB API communication via remote API (ZMQ)
  • ✅ Inverse kinematics solver for 6-DOF manipulator
  • ✅ PID trajectory tracking with visualisation
  • ✅ Path planning with obstacle avoidance (sampling-based)

Technical Implementation

MATLAB Control Architecture: Custom MATLAB script that:

  • Establishes ZMQ socket connection to CoppeliaSim
  • Reads joint angles and end-effector pose at 50 Hz
  • Computes inverse kinematics using damped least-squares
  • Sends velocity commands back to simulation
  • Logs all data for post-processing and visualisation

Kinematics Validation: Compared forward kinematics computed in MATLAB against CoppeliaSim's internal engine – error < 0.01 mm for reachable workspace.

Outcomes & Learnings

  • Control performance: Settling time under 0.5s for step response (PID tuned via Ziegler-Nichols)
  • IK accuracy: Average end-effector pose error = 0.8 mm within workspace
  • Simulation speed: Real-time factor 0.95 on standard laptop
  • Direct application: This pipeline is transferable to real robotic arms (e.g., Dobot, UFactory)
